regridding¶

find_destination_grid(indexes[*]:integer, var:string, field:string, opt[1]:string)¶
Parameters:
  • indexes (integer) – a 1-D array of model indexes to be considered.
  • var (string) – variable name.
  • field (string) – field type.
  • opt (string) – type of grid to be selected: “coarsest”: returns the lowest resolution grid. “finest”: returns the highest resolution grid.
Return value
A 2-D or 3-D dummy variable representing the grid with the attached plev/lat/lon or lat/lon coordinates.
Description:
Given an array of models, returns the coordinates of the coarsest or finest grid, to be used as a destination grid in regridding routines. For the vertical coordinate, the extent is considered as first priority (to avoid loss of data). All models are expect to have the same rank and dimension sizes.
Caveats
The returned plev, lat, lon coordinates do not necessarily come from the same input model, i.e. it is possible to get latitude from one model longitude from another model.
